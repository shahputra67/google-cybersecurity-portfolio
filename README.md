# google-cybersecurity-portfolio

## Cybersecurity Portfolio Activities

### Draft a Professional Statement  
Developed a professional statement to introduce my background, values, and interest in cybersecurity. The statement highlights my strengths, dedication to digital security, and how my technical and analytical skills support organizational security goals.

---

### Conduct a Security Audit  
Performed an internal security audit for a fictional company, Botium Toys, using the **NIST Cybersecurity Framework (CSF)**. The audit assessed risks, identified vulnerabilities, and recommended measures to strengthen security controls and maintain compliance. This activity demonstrates knowledge of risk management, compliance, and audit documentation.

---

### Apply the NIST CSF to Respond to a Security Incident  
Created a detailed cybersecurity incident report guided by the **NIST Cybersecurity Framework**. Analyzed network security events, documented findings, and outlined response strategies to reduce risk impact. This activity shows the ability to handle security incidents through structured frameworks and evidence-based reporting.

---

### Use Linux Commands to Manage File Permissions  
Demonstrated experience managing file permissions through Linux command-line operations. Documented command usage and their effects on system security. This exercise highlights practical knowledge of access control, file management, and permission hardening in Linux environments.

---

### Analyze a Vulnerable System for a Small Business  
Conducted a vulnerability assessment for a small business network. Evaluated security weaknesses, prioritized risks, and proposed remediation steps to prevent potential attacks. This activity reflects proficiency in vulnerability analysis, risk prioritization, and remediation planning.

---

### Document an Incident with an Incident Handler’s Journal  
Maintained an incident handler’s journal throughout multiple cybersecurity scenarios. Recorded incident details, actions taken, and lessons learned. This document demonstrates consistent use of documentation and reflection practices in the incident response process.

---

### Finalize the Incident Handler’s Journal  
Reviewed and refined journal entries to create a comprehensive incident response record. Summarized reflections on tools, techniques, and knowledge gained from hands-on security labs. This final document showcases professional growth and readiness for real-world cybersecurity roles.

---

### Update a File through a Python Algorithm  
Developed a Python algorithm to open, read, and parse file contents. Documented code structure and logic for portfolio inclusion. This project shows understanding of automation in cybersecurity tasks, file analysis, and Python programming for data processing.

---

#### Each of these activities demonstrates applied cybersecurity knowledge and hands-on experience across risk management, system auditing, incident response, Linux administration, and programming. Together, they form a complete portfolio aligned with industry standards and practical job requirements.
